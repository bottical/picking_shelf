1. 作業全体の流れ

シェルフローは、CSVまたはExcelで取り込んだ作業データをもとに、商品の投入先・取り出し先・仕分け先を画面に表示する作業支援システムです。

通常のピッキング支援で使う流れ

  1. HOME画面にログインします。
  2. 投入モードでCSVまたはExcelファイルを読み込みます。
  3. 商品JANをスキャンします。
  4. Wall画面で投入先の空き枠をタップします。
  5. ピッキングモードでピッキングNo.を入力します。
  6. Wall画面に表示された対象間口から商品を取り出します。
  7. 取り出し後、完了操作を行います。

卸仕分けで使う流れ

  1. 卸仕分けモードの取込画面でCSVまたはExcelファイルを読み込みます。
  2. PCスキャン画面で商品バーコードをスキャンします。
  3. 仕分け先表示画面に、卸先ごとの投入数量が表示されます。
  4. 表示された数量を投入し、完了した仕分け先をタップします。
  5. 誤って完了した場合は、完了表示を長押しして取消します。
  6. 必要に応じて簡易実績CSVを出力します。

推奨環境

  • 最新版のGoogle ChromeまたはMicrosoft Edgeを推奨します。
  • バーコードリーダーを使用する場合は、読み取り先の入力欄にカーソルがあることを確認してください。
  • 通信が不安定な場合、CSV/Excel取込や完了状態の反映に時間がかかることがあります。

2. ログインとユーザー選択

ログイン方法

  1. HOME画面を開きます。
  2. メールアドレスとパスワードを入力します。
  3. ログインボタンを押します。
  4. ログイン後、各モードへ進むことができます。

User 1〜User 4について

シェルフローでは、1つのログインアカウントの中で、作業者を User 1〜User 4 として切り替えて利用できます。画面上部のユーザー選択欄で、現在操作するユーザーを選択します。

ユーザーごとに保持される内容

  • 現在作業中のピッキングNo.
  • ピッキングの進捗状態
  • 投入待機中の商品情報

注意点

  • ユーザーを切り替えると、そのユーザーに紐づく作業状態が表示されます。
  • 別ユーザーを選んだまま操作すると、意図しない完了登録や状態変更につながる場合があります。
  • 作業前に、画面上部の選択中ユーザーを確認してください。

3. HOME画面

HOME画面は、ログイン後に最初に表示される画面です。各モードへの移動、ログアウト、データリセットを行います。

主なメニュー

  • 投入モード：CSV / Excelを読み込み、JANスキャンで商品を間口へ割り当てます。
  • モバイルウォール：間口の状態表示、投入先選択、ピッキング対象確認に使用します。
  • ピッキング：ピッキングNo.単位で取り出し作業を進めます。
  • 卸仕分けモード：卸先・店舗・納品先ごとの仕分け数量を表示します。
  • ヘルプ・操作マニュアル：このページを表示します。
  • ログアウト：現在のアカウントからログアウトします。
  • データをリセット：現在の作業データを初期化します。実行前に、対象範囲を確認してください。

データリセットの注意

データリセットは管理用の初期化機能です。現在の投入状態、ピッキング状態、読み込み済みリストなどがクリアされます。

  • 作業途中で実行すると、進行中データが失われる可能性があります。
  • 通常作業者は、管理者の確認なしに実行しないでください。
  • リセット前に、必要な進捗や実績CSVを出力済みか確認してください。
  • 通常ピッキング用データと卸仕分け用データで、リセット対象が異なる場合があります。

4. 事前準備

対応ファイル形式

  • CSV
  • Excel（.xlsx / .xls）

通常ピッキング支援で必要な基本項目

  • ピッキングNo.
  • JANコード
  • 数量

卸仕分けモードで必要な基本項目

  • バーコード列
  • 仕分け先名列
  • 数量列

任意で使用できる項目

  • 商品表示名列
  • 仕分け先コード列

列番号は画面上の設定から変更できます。列番号は左から数えた番号で指定します。A列は1、B列は2、C列は3です。

5. 投入モード

投入モードの役割

投入モードでは、商品JANを読み取り、対象商品をどの間口へ入れるかを登録します。PC側では全体レイアウトを確認しながら操作します。

ファイル読込手順

  1. 投入モードを開きます。
  2. 「ファイルを選択」でCSVまたはExcelファイルを選択します。
  3. 必要に応じて「設定」で列設定を確認します。
  4. 「インポート実行」を押します。
  5. 読み込み完了後、投入対象SKUとピッキングリストが作成されます。

商品投入の流れ

  1. 商品JANをスキャンします。
  2. 対象商品であれば、投入待機状態になります。
  3. Wall画面に「TAP」と表示された投入可能な空き枠が表示されます。
  4. 投入先にしたい空き枠をタップします。
  5. 商品がその間口に登録されます。

スキャン時の判定

  • リストに存在しない商品：エラー表示になります。
  • 既に投入済みの商品：警告表示となり、既に投入されている間口がハイライトされます。
  • 数量1の商品：通常の開始音が鳴ります。
  • 数量複数の商品：複数開始音が鳴ります。

スキャン取り消し

投入待機中は「スキャンを取り消す」でキャンセルできます。キャンセル後は再度JANスキャンが可能になります。

SKUの解除

投入済み間口をクリックすると、その間口に入っているSKU一覧が表示されます。必要に応じて個別SKUを解除し、未割り当てに戻すことができます。

6. Wall画面

Wall画面の役割

Wall画面は、間口ごとの状態表示と操作を行う画面です。投入作業時とピッキング作業時の両方で使用します。

初期設定

初回利用時は、必要に応じて以下を設定します。

  • 総間口数
  • 表示モード
  • 画面の向き
  • 表示サイズ
  • 一覧の行数・列数
  • 他ユーザー表示
  • 一括分割数

表示モード

  • 一覧表示：複数間口を同時に表示するモードです。主にPCやタブレット向けです。
  • 個別表示：1間口ずつ詳細表示するモードです。スマホ向けの利用を想定しています。

投入時の見方

  • 投入待機中は、画面上部の案内バナーに対象JANが表示されます。
  • 投入可能な空き間口には「TAP」が表示されます。
  • 商品を入れたい間口をタップすると、その間口に商品が登録されます。

ピッキング時の見方

  • ピッキング対象間口は色付きで強調表示されます。
  • 表示された数量を取り出してください。
  • 取り出し後、その間口をタップすると完了登録できます。

他ユーザー表示

Wall画面では、他ユーザーが担当している間口にユーザーインジケーターが表示される場合があります。複数人で同時作業を行う際の衝突防止に役立ちます。

分割設定

各間口は最大6分割まで設定できます。分割数の変更は、設定画面または間口別メニューから行います。

その他（未割り当て）

どの間口にも割り当てられていない商品は「その他」に集約表示されます。クリックすると、JANと数量の一覧を確認できます。

7. ピッキングモード

ピッキングモードの役割

ピッキングモードでは、ピッキングNo.ごとに対象商品を表示し、取り出し作業を完了管理します。

作業開始手順

  1. ピッキングモードを開きます。
  2. 作業するユーザーを確認します。
  3. ピッキングNo.を入力します。
  4. Enterで読み込みます。
  5. 対象商品一覧が表示されます。

一覧に表示される内容

  • JAN
  • 数量
  • ロケーション
  • 状態
  • 完了操作

完了操作

  1. Wall画面で対象間口を確認します。
  2. 表示された数量を取り出します。
  3. 通常は、取り出し後に対象間口またはピッキング画面の完了ボタンで完了登録します。
  4. 現場で決められた運用に従い、二重に完了操作しないよう注意してください。
  5. 全て完了すると、一覧が完了状態になり、完了音が鳴ります。

検品モード

検品モードをONにすると、JANを読み取って正しい商品か確認しながらピッキングを進められます。数量検品をONにした場合は、必要数量に達するまでJAN読取ごとに数量が加算されます。

ピッキングリセット

現在のユーザーが実施中のピッキングを解除し、未完了状態をクリアできます。誤って実行しないよう注意してください。

8. 卸仕分けモード

卸仕分けモードの役割

卸仕分けモードでは、商品バーコードをスキャンし、卸先・店舗・納品先ごとに必要な投入数量を表示します。PC側でバーコードを読み取り、スマホやタブレット側で仕分け先ごとの数量を確認して完了登録します。

使用する画面

  • 卸仕分け 取込：CSV / Excelファイルを読み込み、仕分けバッチを開始します。
  • 卸仕分け スキャン：PC側でバーコードを読み取り、現在SKUを切り替えます。
  • 仕分け先表示：スマホやタブレットで、仕分け先ごとの数量を表示します。

取込手順

  1. HOME画面から「卸仕分けモード」を開きます。
  2. CSVまたはExcelファイルを選択します。
  3. バーコード列、仕分け先名列、数量列を指定します。
  4. 必要に応じて、商品表示名列、仕分け先コード列を指定します。
  5. 「取込・バッチ開始」を押します。
  6. 取込結果のプレビューと警告を確認します。
  7. 警告がある場合は、内容を確認して必要に応じて元ファイルまたは列設定を修正してください。

PCスキャン手順

  1. 「PCスキャン画面」を開きます。
  2. バーコード入力欄にカーソルがあることを確認します。
  3. 商品バーコードをスキャンします。
  4. 現在SKUと仕分け先別の進捗が表示されます。
  5. 次の商品をスキャンすると、表示対象SKUが切り替わります。

仕分け先表示画面の使い方

  1. スマホまたはタブレットで「仕分け先表示画面」を開きます。
  2. 端末設定で、表示開始番号と表示件数を設定します。
  3. PC側でSKUがスキャンされると、対象仕分け先に数量が表示されます。
  4. 表示された数量を投入します。
  5. 投入が完了した仕分け先カードをタップします。
  6. 誤って完了した場合は、完了表示を長押しして取消します。

表示数量の見方

  • 卸先ごとの数量を確認し、表示された数量分を投入してください。
  • 「今回投入なし」と表示されている卸先は、対象SKUを投入する必要がありません。
  • 完了後に誤りに気付いた場合は、対象カードの「長押しで取消」表示に従って完了を取り消してください。

直前SKUステータス

PCスキャン画面には、直前SKUの完了状況が表示されます。前SKUに未完了があっても次SKUのスキャンはできますが、未完了が残っていないか確認しながら作業してください。

簡易実績CSV

作業状況を確認したい場合は、PCスキャン画面から簡易実績CSVをダウンロードできます。出力前に、未完了の仕分け先が残っていないか確認してください。

9. よくある質問

バーコードを読み取っても反応しません

  • 入力欄にカーソルが入っているか確認してください。
  • バーコードリーダーが接続されているか確認してください。
  • 別のバーコードで反応するか確認してください。
  • 反応しない場合は、画面を再読み込みしてください。

リストに存在しない商品と表示されます

  • 読み込んだCSV / Excelに対象JANが含まれているか確認してください。
  • JAN列の指定が正しいか確認してください。
  • バーコードに余分な文字や前後スペースが含まれていないか確認してください。

既に投入済みの商品と表示されます

  • 同じJANがすでに別の間口に登録されている可能性があります。
  • 画面上でハイライトされた間口を確認してください。
  • 重複投入が意図したものか、現場ルールに従って判断してください。

「EXCEL読込みライブラリの読込に失敗しました」など、Excel読込エラーが表示されます

  • まず画面を再読み込みしてから再度取り込んでください。
  • 急ぎの場合は、ExcelファイルをCSV形式で保存し直して取り込んでください。
  • 時間をおいても解消しない場合は、ブラウザのキャッシュ削除または別端末で確認してください。
  • 同じ現象が続く場合は、発生日時・使用端末・画面キャプチャを管理者へ連絡してください。

取込後の内容が想定と違います

  • 列番号の指定が正しいか確認してください。
  • 列番号はA列=1、B列=2、C列=3として指定します。
  • 1行目をタイトルとしてスキップする設定が、ファイル内容と合っているか確認してください。

画面が更新されません

  • ブラウザを再読み込みしてください。
  • 通信環境を確認してください。
  • 別端末でも同じ状態か確認してください。
  • 同じ現象が続く場合は管理者へ連絡してください。

完了一覧に表示されません

  • 作業が完了状態になっているか確認してください。
  • 検索条件や日付条件を確認してください。
  • 画面を再読み込みしてください。

間口や仕分け先を誤って完了しました

  • 通常ピッキングの場合は、画面上の状態や操作ボタンを確認してください。
  • 卸仕分けモードの場合は、完了表示を長押しして取消できます。
  • 取消後、正しい数量を確認して再度完了操作を行ってください。

データをリセットしてよいかわかりません

  • 作業中の場合は、原則としてリセットしないでください。
  • リセットすると、現在の投入状態や作業状態が消える可能性があります。
  • 管理者またはシステム担当者に確認してから実行してください。
